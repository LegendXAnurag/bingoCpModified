# Contributing to CP Games

We love your input! We want to make contributing to this project as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## GitHub Workflow
We use GitHub to host code, track issues and feature requests, and accept pull requests.

## Commit Messages
Please follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification for your commit messages.

## General Workflow
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed components or APIs, update the documentation.
4. Ensure the project builds successfully (`npm run build`).
5. Make sure your code lints (`npm run lint`).
6. Submit a pull request!

## Bug Reports
We use GitHub issues to track public bugs. Report a bug by opening a new issue.

**Great Bug Reports** tend to have:
- A quick summary and/or background
- Steps to reproduce
- What you expected would happen
- What actually happens
- Notes (possibly including why you think this might be happening, or stuff you tried that didn't work)
